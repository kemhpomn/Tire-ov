*   **数据集划分**：

    *   **测试集 (Test Set)**：5,000 张正常图像 + 2,000 张异常图像（用于评估模型，寻找最佳 F1-Score 阈值）。

**2. 数据预处理（论文提分关键点 🌟）**
DINOv3 是在 1.70 亿张自然图像（RGB 3通道）上预训练的[2, 3]，而你的 X 光图是单通道灰度图。
*   **尺寸调整**：严格 Resize 到 `256 x 256`（DINOv3 对此分辨率有原生优化[4]）。
*   **通道复制**：将 1 通道复制为 3 通道，否则模型无法接受输入。
*   **均值方差归一化**：不要盲目使用 ImageNet 的均值方差！**在你的论文中写明：你统计了 5 万张轮胎 X 光图的全局均值和方差，并以此进行归一化。** 这体现了你扎实的工程功底。

### 🧠 第二阶段：加载 DINOv3 骨干网络与特征提取

我们直接使用 Hugging Face 上 Meta 官方开源的轻量级蒸馏版 DINOv3（以 `dinov3-vits16` 为例，参数量小且速度极快 [6]）。

**1. 加载冻结的 DINOv3**
```python
import torch
from transformers import AutoModel
import torchvision.transforms as T

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

# 加载 DINOv3 轻量级版本 (ViT-Small, Patch size 16)
model_name = "facebook/dinov3-vits16-pretrain-lvd1689m"
model = AutoModel.from_pretrained(model_name).to(device)
model.eval() # 冻结网络，坚决不反向传播更新参数

# 定义数据预处理流程
transform = T.Compose([
    T.Resize((256, 256)),
    T.ToTensor(),
    T.Lambda(lambda x: x.repeat(3, 1, 1) if x.shape[0] == 1 else x), # 灰度转RGB
    T.Normalize(mean=[0.5, 0.5, 0.5], std=[0.2, 0.2, 0.2]) # 替换为你算出的轮胎数据集均值方差
])
```

**2. 定义多层特征提取函数**
DINOv3 的深层捕捉语义（轮胎结构），浅层捕捉纹理（帘线排布）。我们需要将**最后几层（如第 8, 10, 12 层）的 Patch Token 提取出来并融合**。

```python
def extract_features(images):
    with torch.no_grad():
        # DINOv3 输出通常包含所有隐藏层的状态
        outputs = model(images, output_hidden_states=True)
        hidden_states = outputs.hidden_states
        
        # 提取最后两层特征（例如第11层和第12层）
        layer1 = hidden_states[-1][:, 1:, :] # 去掉 [CLS] token，只留图像 Patch
        layer2 = hidden_states[-2][:, 1:, :]
        
        # 将不同层的特征在通道维度拼接 (Fusion)
        features = torch.cat([layer1, layer2], dim=-1) # shape:[Batch, Num_Patches, Feature_Dim]
        return features
```

### 🗄️ 第三阶段：构建“正常状态记忆库”（Memory Bank）

这一步是整个算法的灵魂。你需要把 4.5 万张正常图片的特征提取出来。

**1. 提取所有正常图片的特征池**
*   45,000 张图 × (256/16) * (256/16) 个 Patch = **1152 万个特征向量**。
*   如果全存下来，推理时对比太慢。我们需要用到**核心集采样（Coreset Subsampling）**。

**2. 核心集降维（使用 FAISS 或 sklearn）**
核心集采样的目的是：在 1152 万个特征中，选出 1%（约 11 万个）最具有代表性的特征，去除那些长得一模一样的正常帘线特征。

```python
import numpy as np
import faiss

# 假设 normal_features_pool 是你提取出的所有正常特征，形状为[11520000, 768]
normal_features_pool = np.random.rand(11520000, 768).astype('float32') # 这里用随机数据代替

# 1. 采用随机下采样或 K-Center Greedy 算法压缩数据
coreset_ratio = 0.01 
num_samples = int(len(normal_features_pool) * coreset_ratio)

# 简单起见，这里演示基于 K-Means 的核心点提取（论文中可对比随机采样、K-Means、K-Center Greedy的差异）
kmeans = faiss.Kmeans(d=768, k=num_samples, niter=20, verbose=True, gpu=True)
kmeans.train(normal_features_pool)
memory_bank = kmeans.centroids # 最终得到的轻量级正常特征记忆库：[115200, 768]

# 2. 构建 FAISS 索引以便极速检索
index = faiss.IndexFlatL2(768) # 使用 L2 欧式距离
# 如果在 GPU 上跑：res = faiss.StandardGpuResources(); index = faiss.index_cpu_to_gpu(res, 0, index)
index.add(memory_bank)
```

### ⚖️ 第四阶段：测试、评分与阈值标定（使用 2000 张异常图）

现在，系统里已经存储了“完美的轮胎 X 光长什么样”（即 `index`）。接下来，我们要让它去找缺陷。

**1. 异常打分（Anomaly Scoring）**
对于测试集中的图片（5,000 正常 + 2,000 异常），逐张输入网络计算得分。

```python
def calculate_anomaly_score(test_image):
    # 1. 提取测试图特征
    test_feat = extract_features(test_image.unsqueeze(0)).squeeze(0) #[256, 768]
    test_feat_np = test_feat.cpu().numpy().astype('float32')
    
    # 2. 去记忆库中找最近邻 (K=1 或 K=9)
    k = 9 
    distances, _ = index.search(test_feat_np, k)
    
    # 3. 计算每个 Patch 的异常得分 (Patch 距离的平均或最大值)
    patch_scores = np.mean(distances, axis=1) # [256]
    
    # 4. 图像级总得分：取整张图中异常最严重的那个 Patch 的得分
    image_score = np.max(patch_scores) 
    return image_score, patch_scores
```

**2. 利用 2000 张异常图标定最佳阈值（论文极其亮眼的一环 ✨）**
无监督检测最大的痛点是“工业落地没法给阈值”。你拥有 2000 张真实异常图片，这能让你画出极其漂亮的 ROC 曲线和 F1-Score 曲线。

```python
from sklearn.metrics import roc_curve, precision_recall_curve, f1_score

# 假设 y_true 是真实标签 (正常=0, 异常=1)，y_scores 是你计算出的 image_score
fpr, tpr, roc_thresholds = roc_curve(y_true, y_scores)
precision, recall, pr_thresholds = precision_recall_curve(y_true, y_scores)

# 寻找 F1 最大时的阈值
f1_scores = (2 * precision * recall) / (precision + recall + 1e-8)
best_idx = np.argmax(f1_scores)
best_threshold = pr_thresholds[best_idx]
best_f1 = f1_scores[best_idx]

print(f"产线部署最佳判定阈值: {best_threshold}, 此时最高 F1-Score 为: {best_f1}")
```

### 📝 核心文件功能详细说明（请照着这个逻辑写代码）

#### 1. `configs/config.yaml` (配置文件)
将所有会变动的参数写在这里，方便你在写论文时做**消融实验（Ablation Study）**。
```yaml
# config.yaml
model:
  name: "facebook/dinov3-vits16-pretrain-lvd1689m"
  patch_size: 16
  extracted_layers:[11, 12] # 提取哪几层特征融合

data:
  image_size: 256
  # 你在5万张轮胎X光图上统计出的均值和方差 (极其重要)
  mean: [0.485, 0.485, 0.485] 
  std: [0.229, 0.229, 0.229]

coreset:
  subsampling_ratio: 0.01  # 保留 1% 的特征
  knn_neighbors: 9         # 推理时寻找最近的 9 个 Patch
```

#### 2. `src/dataset.py` (数据加载)
*   **职责**：读取 `data/` 目录下的图片。
*   **关键点**：使用 `torchvision.datasets.ImageFolder`，并在 `transforms` 里加上 `T.Lambda(lambda x: x.repeat(3, 1, 1))` 把单通道 X 光灰度图复制为 3 通道，否则 DINOv3 会报错。

#### 3. `src/model.py` (模型封装)
*   **职责**：封装 DINOv3 的加载和前向传播。
*   **关键点**：利用 `output_hidden_states=True` 拿到所有层的特征，去掉分类头（`[CLS]` token），只返回空间 Patch 的特征（例如 `256x256` 图像经过 `patch=16` 切分后，会返回 `256` 个向量）。

#### 4. `src/coreset.py` (核心集提取)
*   **职责**：实现降维逻辑。
*   **关键点**：为了毕业论文的丰满度，你可以在这里实现两种方法供对比：一种是普通的随机采样（Random Subsampling），另一种是 `FAISS K-Means` 聚类。在论文里对比这两种方法对检测速度和精度的影响。

#### 5. 执行脚本：`01_extract_and_build_bank.py`
*   **运行逻辑**：
    1. 读取 `config.yaml`。
    2. 加载 `train/good/` 里的 4.5 万张图片。
    3. 送入冻结的 DINOv3 提取特征，全部存入一个巨大的 NumPy 数组。
    4. 调用 `src/coreset.py` 把特征压缩到 1%。
    5. 构建 `faiss.IndexFlatL2`，并把它保存到 `weights/memory_bank/normal_faiss_index.bin`。
*   **耗时预期**：这一步最花时间，但在跑完保存 `.bin` 文件后，以后的实验就不需要再跑这一步了！

#### 6. 执行脚本：`02_test_and_evaluate.py`
*   **运行逻辑**：
    1. 加载刚才保存的 `normal_faiss_index.bin`。
    2. 遍历 `data/test/` 里的所有图片（正常+各类型异常）。
    3. 提取每张测试图特征，用 FAISS 算 L2 距离，得到每张图的 `anomaly_score`。
    4. 利用 `sklearn.metrics` 算出所有图片的 ROC、AUC、PR 曲线和最佳 F1-Score 阈值。
    5. 将高清的图表保存到 `outputs/metrics/`（这些图直接贴进毕业论文的第四章“实验结果与分析”）。

#### 7. 执行脚本：`03_inference_demo.py` (极其抓眼球的一步)
*   **运行逻辑**：输入一张含有气泡的 X 光图片，代码算出这 $16 \times 16$ 个 Patch 的距离得分，通过双线性插值（Bilinear Interpolation）放大到 $256 \times 256$，然后用 OpenCV 叠加 `cv2.COLORMAP_JET`（把高分区域变成红色）。
*   **目的**：输出直观的“缺陷定位热力图”保存到 `outputs/heatmaps/`，向导师或评审专家证明：你的模型不仅知道轮胎坏了，还精确知道坏在哪！
