# 📊 数据集统计工具使用说明

## 🎯 工具介绍

提供了两个统计数据集均值方差的脚本，适用于不同场景：

### 1️⃣ **statistics_dataset.py** - 快速统计版
- ⚡ 速度快，只计算基本统计量
- 📝 输出文本报告
- ✅ 可直接更新配置文件

### 2️⃣ **analyze_dataset_statistics.py** - 详细分析版（推荐）
- 📊 包含完整的可视化图表
- 📈 生成直方图、箱线图、累积分布图
- 📋 详细的统计报告（偏度、峰度等）
- 🎨 图表可直接用于论文

---

## 🚀 使用方法

### 方法一：快速统计（适合只需要数值）

```bash
cd D:\Tire_Xray_Anomaly_Detection_DINOv3

# 激活环境
conda activate tire_dinov3

# 运行快速统计
python tools/statistics_dataset.py
```

**交互流程**：
1. 选择统计范围（全部数据/训练集/自定义）
2. 等待计算完成
3. 选择是否更新 `configs/config.yaml`

**输出结果**：
- 终端显示：均值、标准差、方差
- 文件保存：`outputs/dataset_statistics.txt`

---

### 方法二：详细分析（推荐，适合论文写作）

```bash
cd D:\Tire_Xray_Anomaly_Detection_DINOv3

# 激活环境
conda activate tire_dinov3

# 运行详细分析
python tools/analyze_dataset_statistics.py
```

**交互流程**：
1. 选择要分析的数据集
   - 选项 1: 全部数据（data/）
   - 选项 2: 训练集正常样本（data/train/good/）← **推荐**
   - 选项 3: 测试集正常样本（data/test/good/）
   - 选项 4: 自定义目录

2. 选择采样比例
   - 如果数据量很大（>5 万张），可以选择采样（如 0.5 = 50%）
   - 默认使用全部数据（1.0）

3. 等待自动完成所有分析

4. 选择是否更新配置文件

## 📊 输出内容详解

### 运行详细分析后，你会得到：

```
outputs/statistics/
├── pixel_distribution_histogram.png      ← 像素值分布直方图
├── pixel_distribution_boxplot.png        ← 箱线图
├── cumulative_distribution.png           ← 累积分布函数图
└── statistics_report.txt                 ← 详细统计报告
```

### 图表说明

#### 1. **像素值分布直方图** (`pixel_distribution_histogram.png`)
- 展示轮胎 X 光图像的像素值分布情况
- 红色虚线：均值位置
- 绿色虚线：中位数位置
- **论文用途**：说明数据集的灰度分布特性

#### 2. **箱线图** (`pixel_distribution_boxplot.png`)
- 展示数据的四分位数分布
- 箱体：Q1 到 Q3（中间 50% 的数据）
- 中线：中位数
- 须线：最小值和最大值（排除异常值）
- **论文用途**：展示数据的离散程度和异常值情况

#### 3. **累积分布函数图** (`cumulative_distribution.png`)
- 展示像素值的累积概率分布
- 可以快速看出某个灰度值以下的像素占比
- **论文用途**：说明数据的整体分布形态

## 📋 统计报告解读

打开 `statistics_report.txt`，你会看到：

### 基本统计量

```
均值 (Mean):          0.xxxxxx
标准差 (Std):         0.xxxxxx
方差 (Variance):      0.xxxxxx
最小值 (Min):         0.xxxxxx
最大值 (Max):         0.xxxxxx
中位数 (Median):      0.xxxxxx
第一四分位数 (Q1):    0.xxxxxx
第三四分位数 (Q3):    0.xxxxxx
```

### 分布形态

```
偏度 (Skewness):      x.xxxxxx
峰度 (Kurtosis):      x.xxxxxx
```

**如何解读**：
- **偏度 > 0**：右偏分布（长尾在右侧，亮像素较多）
- **偏度 < 0**：左偏分布（长尾在左侧，暗像素较多）
- **峰度 > 0**：尖峰分布（比正态分布更陡峭）
- **峰度 < 0**：平峰分布（比正态分布更平缓）

**轮胎 X 光的特点**：
- 通常呈现**双峰分布**（一个峰是背景黑色区域，一个峰是轮胎结构的灰白区域）
- 偏度可能接近 0 或略微负偏（因为背景占比较大）

## 🔧 配置文件自动更新

如果选择更新 `configs/config.yaml`，文件会自动修改为：

```yaml
data:
  mean: [0.xxxxxx, 0.xxxxxx, 0.xxxxxx]  # ← 你数据集的实际均值
  std: [0.xxxxxx, 0.xxxxxx, 0.xxxxxx]   # ← 你数据集的实际标准差
```

**为什么复制 3 份？**
- DINOv3 需要 RGB 三通道输入
- 虽然你的 X 光图是灰度图，但会复制为 3 通道
- 所以三个通道的归一化参数相同

## ⚠️ 常见问题

### Q1: 统计过程很慢怎么办？
**A**: 使用采样功能！在提示"是否使用全部数据"时选择 `n`，然后输入采样比例如 `0.5`（50%）。对于 5 万张图，采样 50% 已经足够准确。

### Q2: 应该统计哪个数据集？
**A**: **推荐统计 `data/train/good/`**（4.5 万张正常训练图）。理由：
- 这是你要构建记忆库的数据
- 仅包含正常样本，分布更纯净
- 用这个数据集的统计值做归一化最合理

### Q3: 为什么不用 ImageNet 的默认均值方差？
**A**: 论文中可以这样写：
> "由于轮胎 X 光图像的成像原理与自然图像存在显著差异，直接使用 
> ImageNet 的统计参数会导致归一化后的数据分布偏离预训练模型的期望输入。
> 因此，本文基于 5 万张轮胎 X 光实测数据，重新统计了归一化参数，
> 以确保模型能够提取到更具判别性的特征。"

### Q4: 我的数据集中有很多不同类型的缺陷图像，要不要分开统计？
**A**: **不需要**。原因：
- 无监督异常检测只用正常样本建库
- 归一化参数应该基于正常样本计算
- 缺陷样本的分布会偏离正常样本，这正是异常检测的原理

## 💡 进阶技巧

### 对比不同子集的分布
你可以分别统计：
- `data/train/good/` - 训练集正常样本
- `data/test/good/` - 测试集正常样本
- `data/test/anomaly_*/` - 各类缺陷样本

对比它们的均值方差差异，这本身就是一个有趣的分析点！

### 在论文中展示
将正常样本和缺陷样本的分布直方图画在一起，可以直观展示：
- 正常轮胎的灰度分布相对集中
- 缺陷区域的灰度值往往偏离正常分布

这为你的异常检测方法提供了直观的理论支撑！
